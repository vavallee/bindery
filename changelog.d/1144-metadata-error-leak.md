### Security
- **Metadata provider errors no longer leak the Google Books API key or internal infrastructure** (#1144) — a transport failure while talking to a metadata provider used to echo the raw error back to the client, and that error embedded the full upstream request URL (including the Google Books `?key=` API key) plus the internal DNS resolver IP. The search/series endpoints now return a generic `metadata provider unavailable` message, logging the full detail server-side instead, and the provider clients redact secret query-string values from any error they construct.
