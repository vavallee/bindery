### Added
- **Telemetry: coarse error-class counters** — the once-daily anonymous ping now includes an `errors` section: the number of ERROR and WARN log entries over the last 24 hours, plus up to 5 `{msg, count}` entries for the most frequent errors. `msg` is the fixed, developer-written log message string only (truncated to 120 chars); log details (attrs), which can carry titles, paths, or URLs, are never sent. This surfaces topology-specific breakage (e.g. loopback-Prowlarr SSRF rejections) in aggregate without waiting on bug reports. The existing opt-outs apply unchanged: `telemetry.enabled: false` in Settings → General or `BINDERY_TELEMETRY_DISABLED=true` disables the entire ping, error counters included.
