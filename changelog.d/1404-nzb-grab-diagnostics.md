### Fixed
- **NZB grab failures now explain themselves — including the NZBFinder "error 203" case** (#1404) — when an indexer refuses the NZB download, the error now surfaces the parsed newznab code and description instead of raw XML, and when the grab was redirected off its original host (Prowlarr's per-indexer **Redirect** setting handing the fetch to an app-whitelisting indexer that rejects Bindery's identity) the error names both hosts and points at the Prowlarr setting to disable. Applies to both SABnzbd and NZBGet grabs; a matching entry was added to the troubleshooting guide.
