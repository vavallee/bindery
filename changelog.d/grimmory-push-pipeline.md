### Added
- **Grimmory push pipeline** (#826) — the Grimmory integration now actually pushes books. Every successful ebook import is uploaded to Grimmory's BookDrop inbox (where Grimmory's own metadata matching and review flow takes over), and **Settings → Grimmory** gains a "Push all to Grimmory" bulk sync with live progress for existing libraries. Bindery signs in with a Grimmory username/password (new fields; current Grimmory has no API tokens — the token field remains for when upstream ships them, and takes precedence when set). Pushes are tracked per file (migration 059), so imports and re-run syncs never create duplicate BookDrop entries, and a Grimmory failure never blocks the underlying import — failures log at WARN and surface in the sync status. Test Connection now performs a real login, and the "configuration only" banner is gone.
